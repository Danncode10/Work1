## .env.example Detailed Explanation

This file serves as a template for configuring the application's environment variables. Each variable is explained below:

### Backend Database
```
DATABASE_URL=postgresql://username:password@hostname:5432/dbname
```
- **Purpose**: Connection string for the PostgreSQL database
- **Format**: `postgresql://[user]:[password]@[host]:[port]/[database]`
- **Usage**: Used by FastAPI backend to connect to AWS RDS PostgreSQL instance
- **Example**: `postgresql://myuser:mypass123@myrds.xxxx.us-east-1.rds.amazonaws.com:5432/naturalhealthdb`

### AWS Configuration
```
AWS_REGION=your-aws-region
AWS_ACCESS_KEY_ID=your-access-key-id
AWS_SECRET_ACCESS_KEY=your-secret-access-key
```
- **AWS_REGION**: Specifies which AWS region services are deployed in (e.g., `us-east-1`, `eu-west-1`)
- **AWS_ACCESS_KEY_ID**: Your AWS access key ID for programmatic access
- **AWS_SECRET_ACCESS_KEY**: Your AWS secret access key (keep this secure!)
- **Usage**: Required for AWS SDK/API calls to services like RDS, Cognito, S3

### AWS Cognito (Authentication)
```
AWS_COGNITO_USER_POOL_ID=your-cognito-user-pool-id
AWS_COGNITO_CLIENT_ID=your-cognito-client-id
```
- **AWS_COGNITO_USER_POOL_ID**: Unique identifier for your Cognito User Pool
- **AWS_COGNITO_CLIENT_ID**: App client ID within the User Pool
- **Usage**: Enables user authentication for the platform (optional for browsing)

### Node / FastAPI Backend
```
NODE_ENV=development
PORT=8000
```
- **NODE_ENV**: Environment mode (`development`, `production`, `test`)
- **PORT**: Port number for the FastAPI backend server to listen on
- **Usage**: Controls server behavior and port binding

### Frontend
```
REACT_APP_API_BASE_URL=http://localhost:8000/api
```
- **Purpose**: Base URL for API calls from the React frontend
- **Development**: `http://localhost:8000/api` (points to local FastAPI server)
- **Production**: Would point to deployed backend URL
- **Usage**: Allows frontend to communicate with backend APIs

## How to Use .env.example

1. Copy `.env.example` to `.env` in your project root
2. Replace placeholder values with actual credentials
3. Never commit `.env` to git (it's ignored by `.gitignore`)
4. For team members: Share actual values securely, don't put secrets in repo
